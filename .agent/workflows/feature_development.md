---
description: Standard workflow for developing a new feature
---

1. [Analysis] Analyze the user's request and the current codebase.
2. [Task] Create or update `task.md` to break down the feature into steps.
3. [Plan] Create `implementation_plan.md` detailing the changes (Modified/New/Deleted files) and verification steps.
4. [Review] Ask the user to review the plan.
   // Stop here and wait for user feedback
5. [Implementation] Execute the changes according to the plan.
6. [Verification] Verify the changes (compile, run tests).
7. [Walkthrough] Create `walkthrough.md` to document the result.
